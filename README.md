# AI薬歴アシスタント（CustomTkinter）

<img width="600" height="919" alt="スクリーンショット 2026-03-13 14 46 38" src="https://github.com/user-attachments/assets/ae3fc425-4f35-4175-9f08-e28d9647dd5b" />

薬局現場での**服薬指導メモ作成〜薬歴転記**を効率化するために開発した、
デスクトップ向け業務支援アプリです。

録音した会話内容を Gemini で要約し、SQLite で履歴管理、さらに既存の調剤システムへの自動ペーストまでを一気通貫で行います。

---

## 1. ポートフォリオ要約

- **プロジェクト名**: AI薬歴アシスタント
- **開発目的**: 薬歴作成における記録・要約・転記の工数削減
- **開発形態**: 個人開発
- **想定ユーザー**: 薬剤師 / 薬局スタッフ
- **アピールポイント**:
  - UI・業務ロジック・外部連携を分離した設計（Controller / Services）
  - 音声入力〜AI要約〜DB保存〜自動転記までの業務フローを実装
  - 実務ドメイン（薬歴運用）を前提にした見出しフォーマットと操作性

---

## 2. 解決したい課題

薬局業務では、患者との会話内容を薬歴に整理・記録する工程が負担になりやすく、
以下の課題が発生しがちです。

- 記録漏れや要点の抜け
- 書式統一の難しさ
- 転記作業の手間・入力ミス

本アプリでは、
**録音データから定型フォーマットの要約を自動生成し、履歴管理と転記まで支援**することで、
業務品質と作業効率の両立を目指しています。

---

## 3. 主な機能

1. **録音機能**
   - 服薬指導の会話を録音
   - 録音時間の上限管理

2. **Gemini要約機能**
   - 音声文字起こし結果をもとに薬歴向けフォーマットへ要約
   - 必須見出し（例: 服薬状況 / 副作用 / S/O / A/P / 計画）を意識した整形

3. **履歴管理（SQLite）**
   - 名前・日付・メモで要約データを保存／検索
   - 過去データの再表示・編集・上書き保存

4. **自動転記機能**
   - 調剤システム画面を対象に、見出しごとにコマンド入力＋ペーストを自動化
   - 手作業での転記負荷を削減

---

## 4. 使用技術

- **言語**: Python
- **UI**: CustomTkinter
- **AI**: Google Gemini API（`google-genai`）
- **音声処理**: `sounddevice`, `soundfile`, `numpy`
- **自動化**: `pyautogui`, `pyperclip`, `pygetwindow`
- **DB**: SQLite
- **その他**: `threading` を利用したUI応答性の確保

---

## 5. 設計・実装上の工夫

- **関心の分離**
  - `ui/`: 画面表示・ユーザー操作
  - `controller/`: 画面と業務ロジックの橋渡し
  - `services/`: 音声録音・AI連携・DB操作・自動転記などの処理実体

- **運用を想定したデータ管理**
  - 日付・名前単位での要約検索
  - 要約本文だけでなく文字起こしも保持し、振り返り可能

- **業務フォーマット準拠**
  - セクション見出しを解析し、調剤システムへの転記単位を統一

---

## 6. 実行方法（ローカル）

- windows環境にてpyinstallerでexe化
- exeファイルと`データ保管場所.txt`を同一フォルダに配置し、起動
- DB、prompt.txtの保管場所のパス(サーバ上を想定)を`データ保管場所.txt`に記載し、複数端末から同じ要約記録にアクセス

### 事前準備

- Gemini APIキーなど、実行環境に必要な設定
- `prompt.txt` のプロンプト調整（運用方針に合わせて編集可能）
